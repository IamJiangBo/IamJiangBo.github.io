WSL2的安装教程

发表于 2022-06-26 更新于 2022-06-27 分类于 技术

WSL2介绍

WSL2：使用与linux的windows子系统。接触WSL2，源自于VMware对于系统的资源耗费甚多，由于在windows笔记本上需要经常用到linux，所以转战WSL2。

安装教程

  1. 建议按照windows官网给出的安装教程做。值得注意的是，我按照自动安装：wsl —install 并没有任何反应，最后通过手动安装才成功。
    • 自动安装教程： 安装 WSL | Microsoft Docs
    • 手动安装教程：旧版 WSL 的手动安装步骤 | Microsoft Docs
  2. 安装之前可能需要在原来的windows上开启一些选项，详细见： 适用于 Linux 的 Windows 10/11 子系统（WSL2）；
  3. 在安装之后ubuntu默认安装在C盘，这使得C盘本就捉襟见肘的我来说，需要寻找一种能够使WSL2挪到其他盘的方法，具体参见： wsl2安装教程以及修改默认安装目录到其他盘；这个博客只看最后wsl2改变系统位置即可；

使用教程

  1. 在使用ubuntu系统时，难免会用到root用户，但是ubuntu默认情况下root用户的密码是随机的，我们需要设置root密码。具体参见：wsl设置root密码；

  2. 我们需要在新的wsl环境中安装很多工具，比如Miniconda 或者 Anaconda，但是下载极其缓慢。这个时候我们需要在本地下载好上传到wsl的ubuntu中。这里其实有两种做法：

    • 一种是直接将下载好的工具包在windows资源管理器中拷贝到wsl的文件系统中（因为wsl的linux的目录在windows资源管理器中可以看见）；

    • 另一种是通过scp，这是一种比较常用的方法，这种方法通过ssh实现。首先，在wendows中安装ssh客户端：windows 10安装ssh服务端和客户端并启用；其次在linux中安装ssh服务端：ubuntu安装ssh服务器。然后，再windows中使用ssh-keygen -t rsa 生成ssh的公钥和密钥（一般在当前用户下会生成一个.ssh我文件夹，其中后缀为.pub的为公钥），在linux中做同样操作（并手动在/home/xxxx（这里是我的用户名，用自己当前的linux用户名即可）/.ssh文件夹下创建文件：authorized_keys），将windows的.ssh文件夹中的id_rsa.pub的内容（建议使用Notepad++打开id_rsa.pub文件）拷贝到authorized_keys文件中。最后使用scp语法把本地文件传至linux。例如：scp C:\Pytorch常用函数手册.pdf username@ip:Documents/

      注意：无论是使用ssh username@ip登录linux还是使用scp 本地文件地址 username@ip:文件夹/文件夹/ 都要确保linux上的ssh服务处于开启状态。开启ssh服务用： sudo service ssh start

  3. 如果需要在linux中使用GPU做计算，需要在liunx中安装cuda。而且在linux中安装cuda前在windows系统中更新一下nvidia的驱动。这里有一个坑就是不要用电脑自带的安全管家之类的更新nvidia驱动，我的就是遇到这个坑，我的本是拯救者，用自带的联想电脑管家始终检测到当前驱动是最新的驱动，但是官方的最新驱动比联想电脑管家检测到的高好几个版本。推荐下载nvidia他们家的：GeForce Experience工具检测更新nvidia驱动，下载地址：GeForce Experience（简称 GFE）自动更新驱动并优化游戏设置 | NVIDIA。然后，在linux的命令行中按照nvidia网站给出的方法安装cuda：CUDA Toolkit 11.7 Downloads | NVIDIA Developer。（值得注意的是nvidia给出的安装方法中的cuda版本始终是和他们的最新驱动对标的，这就是为什么要在安装cuda前，要将自己的nvidia驱动更新到最新）。安装完之后可以使用如下命令检测否安装成功：
